# 1. TESTY ROBOTS.TXT:
#    - Validator: https://www.robotstxt.org/
#    - Google: https://search.google.com/search-console (Tools > robots.txt tester)
#    - Bing: https://www.bing.com/webmasters/

# 2. BEST PRACTICES:
#    - Zawsze używaj HTTPS w linach Sitemap
#    - Robots.txt jest CASE-SENSITIVE (wielkie litery mają znaczenie)
#    - Czytaj od góry - pierwsza pasująca reguła wygrywa
#    - User-agent: * dotyczy WSZYSTKICH botów (z wyjątkami poniżej)
#    - Disallow: / = blok całej strony (niechcemy tego!)
#    - Allow: i Disallow: dotyczą ścieżek (zaczynają się od /)

# 3. BEZPIECZEŃSTWO:
#    - Robots.txt jest PUBLICZNIE dostępny (bądź ostrożny z informacjami)
#    - Nie polega się na nim do ochrony poufnych danych
#    - Dla rzeczywistej ochrony użyj .htaccess lub authorization

# 4. AKTUALIZACJA:
#    - Po zmianach w strukturze strony - aktualizuj robots.txt
#    - Poinformuj Google w Search Console
#    - Czekaj 24 godzin na zmianę w indeksie

# 5. LINKI:
